VITAL ESP32 firmware layout
Map of the Arduino sketch: what each file is for, how the modes map to runners,
and the conventions that dictate the folder shape. Written for a human reader.

COMPILE MODEL
-------------
Header-only. There are no .cpp files anywhere. esp32.ino includes exactly one runner
header, which includes the rest of src/, so the whole sketch is a single translation
unit. Every class lives entirely in one .h with all of its methods defined inside the
class body (implicitly inline), which is ODR-safe because there is only one translation
unit. Every header starts with #pragma once.

TREE
----
esp32.ino                     Thin main: picks ACTIVE_MODE, instantiates one runner, delegates setup/loop.
secrets.h                     Real Wi-Fi and backend credentials (gitignored, never committed).
secrets.example.h             Template to copy to secrets.h; documents the four required defines.
pins.txt                      Board terminal map and the VITAL wiring plan.
structure.txt                 This file.
.gitignore                    Keeps secrets.h out of the repo.

src/config/
  Pins.h                      Single home for every pin number and board wiring constant, with terminal comments.

src/hardware/
  Lcd.h                       16x2 I2C display wrapper (hd44780), padded no-flicker write, safe no-op when absent.
  TemperatureProbe.h          DS18B20 async probe, keeps the last good reading, NAN-guards -127 and 85.
  EnergyMeter.h               PZEM-004T v3 wrapper, reads voltage/current/power/energy/frequency/pf as a struct.
  Relay.h                     Single relay output with configurable active level, tracks closed/open state.

src/net/
  BackendClient.h             HTTPS calls to the backend: postReading and postHeartbeat.
  WifiLink.h            Wi-Fi lifecycle: connects to the fixed compiled Wi-Fi with a bounded timeout.

src/core/
  Monitor.h                   Overload state machine (threshold, trip, reclose) plus serial publish and LCD screen.

src/tests/
  Ds18b20Test.h               Runner: enumerate the OneWire bus and print raw temperature.
  LcdTest.h                   Runner: bring up the LCD and cycle through test screens.
  NetTest.h                   Runner: connect and post a temperature reading to the backend.
  (core/Main.h)               The real firmware runner: relay always on, reads temp + PZEM, posts to the app, heartbeat.
  PzemTest.h                  Runner: read the PZEM every 2s and toggle the relay every 10s, printing both over serial.

MODES
-----
Selection is a compile-time #define ACTIVE_MODE in esp32.ino. All src/ code compiles in
every mode; only the chosen runner is instantiated and run. Suggested flash order after wiring:

Value               Runner class    Purpose
MODE_LCD_TEST       LcdTest         Flash first after wiring to confirm the display is detected and readable.
MODE_DS18B20_TEST   Ds18b20Test     Confirm the temperature probe is on the bus and returning sane values.
MODE_PZEM_TEST      PzemTest        Confirm the PZEM reads over UART and the relay toggles on its pin.
MODE_NET_TEST       NetTest         Confirm backend connectivity: Wi-Fi join and reading upload.
MODE_MAIN           Main            The real firmware (src/core/Main.h): relay stays on, reads temp + PZEM, posts readings to the app, sends a device heartbeat.

MODE_NET_TEST is the active mode as shipped.

COMPONENTS
----------
Lcd
  Owns the hd44780 I2C driver. begin() starts the bus on the custom SDA/SCL pins and lets
  hd44780 auto-detect the backpack address; present() and status() report the result; show(line1,
  line2) overwrites both rows padded to width with no flicker and is a safe no-op when no LCD is
  wired. formatFloat(value, digits) is a static helper that renders NAN as "--".

TemperatureProbe
  Owns a OneWire bus and DallasTemperature sensor on one pin. begin() enables the pull-up and starts
  the async (no-wait) conversion cycle. read() returns the previous conversion, requests the next,
  and keeps the last good value so a -127 or 85 glitch never overwrites a real temperature.

EnergyMeter
  Wraps a PZEM-004T v3 on a hardware UART. begin() opens the port at 9600 8N1 on the RX/TX pins.
  read() returns a Reading struct with voltage, current, power, energy, frequency, and powerFactor.

Relay
  Owns one output pin with configurable on/off levels (active-low as wired). begin() sets the pin to
  output and closes the relay. set(closed) drives it; isClosed() reports the tracked state.

WifiLink
  Owns the Wi-Fi lifecycle for a fixed board. connect() joins the compiled WIFI_SSID/WIFI_PASSWORD
  from secrets.h with a bounded 15s timeout, and shows the LCD "WiFi connecting" then the SSID and IP
  on success or "No WiFi" on failure. No NVS store, no multi-network chain, no captive portal, no
  config sync: the board always targets the one Wi-Fi it was compiled with. Takes only an Lcd.

BackendClient
  Stateless HTTPS client for the backend, keyed by the x-device-key header. postReading(...) builds
  a partial JSON body that omits any NAN field and POSTs it to /api/v1/readings. postHeartbeat()
  reports device telemetry and reads the alarm thresholds back out of the response, which Main
  persists to NVS and hands to Monitor so an operator threshold change reaches the board.

Monitor
  Takes references to EnergyMeter, TemperatureProbe, Relay, and Lcd. begin() brings up all four.
  loop(now) samples on interval, runs the threshold/trip/reclose state machine, publishes a JSON
  line over serial, and refreshes the LCD. testLoop(now) is the relay-test variant that toggles the
  relay on a timer and publishes without the state machine. setTestMode() flags the published output.
  snapshot() returns the latest readings for the networking layer to post.

CONVENTIONS
-----------
Compile scope: the Arduino IDE compiles the sketch at the root plus any .cpp under src/. This sketch
  has no .cpp: every class is defined inline in its header and pulled in through esp32.ino, so the
  build is a single translation unit. That is why inline-in-class definitions are ODR-safe here.
Class idiom: each class owns its hardware and state, takes pins/config through its constructor, and
  follows the Arduino begin()/loop() pattern. No free functions live outside esp32.ino.
Shared instances: no globals are defined in the src/ headers. Each runner constructs exactly the
  objects it needs as members, so hardware a mode does not use is simply never begun. The only global
  is the single `runner` in esp32.ino for the active mode.
Secrets: credentials live in gitignored secrets.h, copied from secrets.example.h. Files under src/net
  reach it as ../../secrets.h. It defines WIFI_SSID, WIFI_PASSWORD, BACKEND_URL, DEVICE_KEY.
Test twins: each core/hardware class has a runner twin under src/tests that exercises it in isolation.
Libraries to install (Library Manager):
  hd44780 by Bill Perry      16x2 I2C LCD, auto-detects the PCF8574 backpack address and mapping.
  OneWire                    DS18B20 bus.
  DallasTemperature          DS18B20 temperature conversion.
  PZEM004Tv30                PZEM-004T v3 energy meter.
Bundled with the ESP32 core (no install): WiFi, Wire, WiFiClientSecure, HTTPClient, HardwareSerial.
